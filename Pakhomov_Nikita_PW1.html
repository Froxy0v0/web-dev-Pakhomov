HTML5 Standard and Its Development

When did HTML5 become a W3C Recommendation?
HTML5 became a W3C Recommendation on 28 October 2014.
What were the main reasons for developing HTML5?
  • Modernize the Web: HTML5 aimed to address the limitations of previous versions (like HTML 4) and provide a more robust, feature-rich platform for web development.
  • Support Rich Media: With native support for audio, video, and canvas elements, HTML5 made it easier to create interactive and multimedia-rich web applications.
  • Enhance Semantics: HTML5 introduced new semantic elements (such as <header>, <nav>, and <article>) to improve the structure and meaning of web content.
  • Improve Accessibility: HTML5 emphasized accessibility features, making it easier for assistive technologies to interpret web pages.
  • Standardize APIs: HTML5 standardized various APIs (like geolocation, local storage, and web workers) to enhance web application capabilities.

Difference Between W3C HTML5 and WHATWG HTML Living Standard

What are the key differences between W3C HTML5 and WHATWG HTML Living Standard?
  • Main Element Usage: W3C restricts the use of the <main> element to one visible instance per document, while WHATWG allows multiple <main> elements in a document.
  • Semantic Elements: W3C provides detailed information on how screen readers interpret semantics for elements like <header>, <footer>, <section>, <aside>, and <article>, whereas WHATWG encourages user agents to provide keyboard navigation for these semantic elements.
  • Main Content Definition: W3C defines <main> as representing the main content of a document, whereas WHATWG defines <body> as representing the content of the document.
  • Title Attribute Usage: W3C discourages the use of the title attribute for image captions, while WHATWG promotes <figcaption> for image captions.
  • Other Differences: Varying recommendations related to list markup in <nav>, use of headings, and more.

Minimal HTML Page Code

What does a minimal HTML page consist of?
A minimal HTML page consists of:
  • HTML declaration: <!DOCTYPE html>
  • HTML root element: <html>
  • Head section: <head>
  • Title: <title>
  • Body section: <body>
  • Main content: e.g., <h1>Hello, World!</h1>

Semantic vs. Presentational Markup

What are the differences between semantic and presentational markup?
Semantic Markup:
  • Focuses on the meaning and structure of content.
  • Uses elements like <article>, <section>, and <nav> to convey context.
  • Enhances accessibility and SEO.
Presentational Markup:
  • Focuses on appearance (e.g., styling with <font> or <b> tags).
  • Deprecated because:
  • It mixes content and presentation, making maintenance difficult.
  • CSS now handles styling, separating content from design.

How can you check browser support for tags/attributes?
Use online resources like MDN Web Docs or Can I Use to check compatibility.

Officially Supported Audio and Video Formats

What are the officially supported audio and video formats?
Audio: Supported formats include MP3, AAC, and Vorbis (Ogg).
Video: Supported formats include MP4, WebM, and Ogg Theora.
If you use an unsupported format, the browser may not play the media or require additional plugins/extensions.
